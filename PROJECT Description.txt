PROJECT Description:

 Multi-Stage Dropper Orchestrator: Step-by-Step Payload Extraction with Automated Stage-Graph Reconstruction
Design an agent that orchestrates static disassembly (Ghidra) followed by controlled step-by-step dynamic execution (x64dbg) to unpack and trace multi-stage droppers. The agent maps inter-stage dependencies, extracts embedded payloads at each transition point, and reconstructs the full stage graph. Inspired by autonomous malware triage research requiring chainable tool selection, the orchestrator operates without manual intervention.
14
Core Tools Ghidra, x64dbg (scripting API), Python orchestration layer, Cuckoo Sandbox Dataset Multi-stage malware: Emotet dropper chains, Trickbot loaders from theZoo Difficulty Advanced
Key Objectives
• Parse dropper stage boundaries via static CFG analysis and import anomalies.
• Automate controlled single-step execution to capture payload drops at each stage.
• Recursively analyse extracted payloads using the same pipeline.
• Produce a stage-dependency graph with SHA-256 hashes, IOCs, and MITRE ATT&CK chain.
