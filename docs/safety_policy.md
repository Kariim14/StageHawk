# Safety Policy

This is a defensive academic project.

Allowed:

- Analyze the included educational fake sample.
- Parse static metadata and runtime logs.
- Copy safe evidence from a controlled Windows VM.
- Build reports, hashes, IOCs, and stage graphs.

Not allowed:

- Adding shellcode execution.
- Adding credential theft.
- Adding persistence, evasion, or spreading behavior.
- Running real malware on the Ubuntu/controller host.
- Enabling real malware execution by default.

Real malware support must remain quarantine/sandbox-only, disabled by default, and documented before use.
